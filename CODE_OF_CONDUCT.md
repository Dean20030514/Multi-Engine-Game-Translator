## 简体中文

本项目采用 **[Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)** 作为社区行为准则。完整中文版见：

> https://www.contributor-covenant.org/zh-cn/version/2/1/code_of_conduct/

### 简要原则

- **尊重所有参与者**：欢迎不同水平 / 背景 / 经验的贡献者参与。
- **建设性沟通**：技术分歧用代码 / 数据 / 文档佐证，避免人身评价。
- **聚焦项目目标**：本项目是单一用途工具（多引擎游戏汉化），讨论保持 on-topic。
- **遵守开源精神**：MIT 许可下贡献者保留作者身份；禁止抄袭未署名上游代码（见 [hard contract #6](CLAUDE.md) — 6 处 `anonymousException` 上游归属永久保留）。

### 适用范围

本准则适用于：

- GitHub Issues / Pull Requests / Discussions
- Commit messages / code reviews
- 所有以项目身份参与的公开活动

### 报告渠道

涉及违规行为：

1. **公开 / 一般问题** — 在 GitHub 开 issue（附上下文 + 链接）
2. **私密 / 敏感问题**（如骚扰、个人安全）— 通过 [SECURITY.md](SECURITY.md) 私密渠道联系 maintainer
3. **GitHub 平台滥用**（spam / 仇恨账号）— 直接 GitHub 官方 [report 流程](https://docs.github.com/en/communities/maintaining-your-safety-on-github/reporting-abuse-or-spam)

**Maintainer 收到报告后会**：在合理时间内私密回复 → 评估 → 视严重程度采取行动（警告 / 删除内容 / 封禁互动 / 上报 GitHub）。所有处理对当事人匿名。

### 决策权

当前项目治理模型：**BDFL（Benevolent Dictator For Life）** — @Dean20030514 是唯一 maintainer，对 PR merge / CoC 执行 / backlog 优先级有最终决定权。详见 [CONTRIBUTING.md "Governance" 段](CONTRIBUTING.md)。

## English

This project adopts **[Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)** as its community code of conduct. Full text:

> https://www.contributor-covenant.org/version/2/1/code_of_conduct/

### Core principles

- **Respect all participants**: Contributors of all skill levels, backgrounds, and experience are welcome.
- **Constructive communication**: Technical disagreements should reference code, data, or documentation — not personal evaluations.
- **Stay on topic**: This is a single-purpose tool (multi-engine game translator); discussions stay focused.
- **Honor open-source attribution**: MIT licensed; contributor authorship preserved. Plagiarism of unattributed upstream code is forbidden (see [hard contract #6](CLAUDE.md) — 6 `anonymousException` upstream attribution sites permanently preserved).

### Scope

This Code of Conduct applies to:

- GitHub Issues / Pull Requests / Discussions
- Commit messages / code reviews
- All public activity carried out under the project's identity

### Reporting

For violations:

1. **Public / general issues** — Open a GitHub issue (with context + links).
2. **Private / sensitive issues** (harassment, personal safety) — Contact the maintainer privately via the [SECURITY.md](SECURITY.md) channel.
3. **GitHub platform abuse** (spam / hate accounts) — Use [GitHub's official report flow](https://docs.github.com/en/communities/maintaining-your-safety-on-github/reporting-abuse-or-spam) directly.

**On receiving a report, the maintainer will**: respond privately within a reasonable timeframe → assess → take appropriate action depending on severity (warning / content removal / interaction ban / escalation to GitHub). All handling is anonymous toward the reporter.

### Decision authority

Current governance model: **BDFL (Benevolent Dictator For Life)** — @Dean20030514 is the sole maintainer with final authority over PR merges, CoC enforcement, and backlog priorities. See [CONTRIBUTING.md "Governance" section](CONTRIBUTING.md) for details.
